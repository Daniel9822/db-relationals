1.¿Que es una base de datos relacional y cuál es su estructura basica? 
  
  - Es una base de datos que organiza los datos en tablas relacionadas. La estructura basica incluye tablas, filas y columnas.

2. ¿Cual es la importancia de la integridad referencial en las bases de datos relacionales?

    - Garantiza la consistencia de los datos al mantener relaciones validas entre las tablas.

3. Cuales son algunas tecnicas comunes de optimización de consultas en bases de datos relacionales?

  - Uso de indices, optimizacion de consultas y desnormalizacion.

4. ¿Que es el lenguaje SQL y cuales son sus principales componentes?

  - SQL es un lenguaje de programación para gestionar bases de datos. Sus componentes son DDL, DML y DCL.

5. ¿Cual es la diferencia entre DDL, DML y DCL en SQL?

  - DDL se utiliza para manipular los datos, DML para definir la estructura y DCL para controlar permisos de acceso.

6. ¿Por que es importante la normalización en el diseño de bases de datos relacionales?

  - Para evitar redundancia y garantizar la integridad de los datos.

7. ¿Como se manejan las transacciones en las bases de datos relacionales?

  - Mediante sentencias SQL como BEGIN TRANSACTION, COMMIT y ROLLBACK.

8. ¿Que ventajas ofrece la replicacion en bases de datos relacionales?

  - Mejora la disponibilidad y la redundancia de los datos.

9. ¿Cual es el papel de los indices en la optimizacion del rendimiento en bases de datos relacionales?

  - Mejoran el rendimiento de las consultas al permitir un acceso mas rapido a los datos.

10. ¿Que factores pueden afectar negativamente el rendimiento de una consulta SQL y como se pueden mitigar?

  - Falta de indices, consultas mal escritas y uso excesivo de funciones. Se pueden mitigar creando índices adecuados y optimizando consultas.
